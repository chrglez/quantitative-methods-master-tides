# Day 1 · Guided exercise

**Topic:** set up your Git/GitHub workflow and take the first steps in R.

## Tasks

1. Fork this repository and clone your fork locally.
2. Open the project in RStudio (double-click the `.Rproj` file).
3. Complete `exercise-template.R` — follow the numbered sections.
4. Run it top-to-bottom to verify it works without errors.
5. Commit `exercise-template.R` and push to your fork.
6. Confirm on `github.com/<you>/quantitative-methods-master-tides`
   that your commit is visible.

## Deliverable

A Pull Request from `your-user:day-1` to `chrglez:main`, with your
`.R` file committed on the branch. Full submission workflow in
[`exercises/README.md`](../README.md).

::: callout-note
For Days 1–5 you submit plain **`.R` scripts**. Quarto documents
(`.qmd`) are introduced in Day 6 and only used for the final graded
report.
:::
